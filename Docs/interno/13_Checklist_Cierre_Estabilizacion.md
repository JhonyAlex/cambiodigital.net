# Checklist de Cierre de Estabilización (Mes 3)

Este checklist debe completarse al final de la fase de estabilización de 90 días para formalizar la transición a la operación normal del servicio IT Partner.

## 1. Verificación de Entregables de Estabilización

Antes de contactar al cliente para el cierre, verifica que todos los objetivos de la estabilización se hayan cumplido.

-   [ ] **Documentación Completa:**
    -   [ ] El `Documento_Inventario_TI.md` está completo y actualizado.
    -   [ ] El registro de accesos y credenciales está guardado de forma segura en el gestor de contraseñas.
-   [ ] **Sistema de Backups Operativo:**
    -   [ ] Los backups automáticos están configurados y funcionando.
    -   [ ] Se ha realizado al menos una prueba de restauración exitosa.
    -   [ ] La política de retención de backups está documentada.
-   [ ] **Sistema de Monitoreo Activo:**
    -   [ ] El cliente está configurado en la herramienta de monitoreo.
    -   [ ] Las alertas críticas están activas y probadas.
    -   [ ] (Opcional) Se ha compartido un dashboard de estado con el cliente.
-   [ ] **Optimización y Seguridad:**
    -   [ ] Se han aplicado las actualizaciones de seguridad críticas.
    -   [ ] Se han implementado las optimizaciones de rendimiento planificadas.
-   [ ] **Capacitación Realizada:**
    -   [ ] Se han impartido las horas de capacitación correspondientes a los primeros 3 meses.
    -   [ ] Los temas y asistentes de cada sesión están documentados.

## 2. Preparación del Reporte de Cierre

-   [ ] **Recopilación de Datos:**
    -   [ ] Reunir los logs, reportes y notas de los últimos 90 días.
    -   [ ] Tomar capturas de pantalla del dashboard de monitoreo, sistema de backups, etc.
-   [ ] **Redacción del Reporte de Estabilización:**
    -   [ ] Utilizar la plantilla `Reporte_Estabilizacion_Template.md`.
    -   [ ] **Resumen Ejecutivo:** Un párrafo que describa el estado "antes y después".
    -   **Trabajo Realizado:** Detallar las acciones completadas en cada área (documentación, backups, monitoreo, optimización).
    -   **Estado Actual del Sistema:** Describir la salud actual de la infraestructura (rendimiento, seguridad).
    -   **Próximos Pasos:** Explicar cómo funcionará el soporte a partir de ahora (revisiones semanales, soporte por ticket, etc.).
-   [ ] **Revisión Interna:** El Gerente de Cuenta o un Técnico Senior debe revisar y aprobar el reporte antes de enviarlo al cliente.

## 3. Ejecución de la Transición

-   [ ] **Agendamiento de la Reunión de Cierre:**
    -   [ ] Invitar al cliente a una llamada de 30 minutos para presentar los resultados.
    -   [ ] Enviar la invitación con una agenda clara.
-   [ ] **Ejecución de la Reunión de Cierre:**
    -   [ ] Presentar el Reporte de Estabilización al cliente.
    -   [ ] Resaltar los logros y el valor entregado durante los 90 días.
    -   [ ] Responder cualquier pregunta que el cliente pueda tener.
    -   [ ] Confirmar verbalmente la transición al modelo de suscripción mensual.
-   [ ] **Envío de Comunicación Formal:**
    -   [ ] Enviar el email de "Fin de Estabilización" (usar plantilla).
    -   [ ] Adjuntar el Reporte de Estabilización en formato PDF.
-   [ ] **Actualización de Sistemas Internos:**
    -   [ ] Marcar la fase de estabilización como "Completada" en el sistema de gestión de clientes.
    -   [ ] Notificar al departamento de finanzas para que la facturación recurrente mensual comience en el siguiente ciclo.
    -   [ ] Archivar este checklist en la carpeta del cliente.
